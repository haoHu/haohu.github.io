构造青蛙

08/08/2015

Don’t Dissect the Frog, Build it

我们都是一路人

古代求道的方士，通过不断的探索，用所积累的学识，追求参悟自身和
天地的真理。

今天的程序猿，通过学习的知识和技巧，追求着用代码，析构世间万物。

两者的经历，出奇地相近。

古代的方士，通过书写，记录下自己对真理的领悟；

今天的程序猿，通过博客，分享自己对编程的感悟。

两者殊途同归。

程序猿们处在一个好时代

现如今，我们有了hexo这样的个人博客利器，同时又拥有github这样无限的空间，剩下的就是我们激发我们写作的热情。

你的博客就像你在虚拟世界中的一个小家，如果你的装修明显与别人不同，有着标新立异的创新，那么在这个同质化泛滥的世界中，很可能会轻松地引来更多的关注。让你在这个世界中大放异彩。自然也就激发你创作更多文章的动力。

为自己的博客装点一番

我们可以设计自己的皮肤，让我们的内容更具个性化，但是设计一套皮肤，我们要去看hexo的代码，修改他的代码，一路下来，往往会打击你的个性化装扮的积极性，到最后往往放弃掉了。那你可能会想，谁还能拯救我呢？

《程序员的思维修炼》里如是说：

Don‘t dissect the frog, build it.—-真正想要了解一只青蛙，传统的解剖不是办法，更好的方法是构造一只青蛙。

在Google上我们会找到很多学习资料，教你如何利用hexo构建一个博客，如何将文章部署到github page上，如何添加插件，等等。至于如何构建一套符合自己风格，量身打造的博客，这个问题就庞大的多了。我将带大家像盖房子一样，搭建一个属于自己的博客。

hexo主题都包含什么

主题的工作就是掌控你的博客到底长什么样子，以及它所具备的功能。

hexo的主题存放路径在你工程目录的themes目录下，当你安装好hexo后，你会看到themes目录下默认会有一个landscape主题。我们暂时不需要搞清楚它里面到底有什么，东西太多了，我们不要解剖它。

我们只需要大概了解它这个主题里都包括什么东西。

我们进入landscpe主题目录，可以看到，它主要包含了

|--Gruntfile.js # 用于构建主题的发布任务
|
|--LICENSE # 版权声明
|
|--README.md # 对主题的介绍和文档    
|
|--_config.yml # 主题模板的配置文件
|
|--layout/    # 模板文件
|
|--package.json # 主题生成需要的第三方插件和工具
|
|--scripts/ # js脚本
|
|--source/ # 主题相关的图片资源，插件资源，脚本文件

下面我们就分别介绍下这些部分都是做什么的。

模板(layout)

这是主题中必要元素。layout中唯一不能缺少的就是index.ejs，它是用来生成首页的。

我们可以想到，一般博客都要有一个文章列表（常在首页看到），点击文章列表的某一篇文章，会跳转到文章页面。有了这两样，至少我们最基本的博客功能就具备了。所以我们最基本的需求就是要有一个文章列表模板和一个文章详情模板

我们可以像婴儿学步一样，利用模仿，通过机械的记忆，慢慢到理解。

我们可以先在themes目录下建立一个与landscape同级的目录–garagekits，模仿landscape，把我们自己主题的目录的结构搭建起来。

需要注意的是，hexo默认使用的主题是landscape，我们要想学习模板，先不要忘记修改一下hexo的_config.yml文件的theme字段，让hexo使用我们自己的主题–garagekits。

我们的主题现在增加了一些文件：

garagekits
 |
 |--layout
 |     |--_partial
 |     |     |-- post_entry.ejs #主页文章列表中每项模板
 |     |
 |     |--index.ejs #网站主页模板，必不可少
 |     |--post.ejs # 每篇单独文章的模板
 |

我们要为每个模板添加些代码

index.ejs:

<head>
<title>index.html</title>
</head>
<body>
    <% site.posts.sort("date", "desc").each(function (post) { %>
        <%- partial("_partial/post_entry", {post : post}) %>
    <% }) %>
</body>

post_entry.ejs

<div>
    <span><%- post.date.format("YYYY-MM-DD") %></span>
    <a href="<%- config.root %><%- post.path %>"><%- post.title %></a>
</div>

post.ejs

<head>
    <title><%- page.title %></title>
</head>    
<body>
    <%- page.title %>
    <hr>
    <%- page.content %>
</body>

其中ejs是一种HTML的模板语言，具体的文档可以点击这里查看。

我们这里主要用到了两种：
1. <%- %> 就是把要写到页面上的内容包含其中
2. <% %> 就是模板运行需要的js代码，它最终不会输出到页面中

我们还用到了hexo的一些全局变量，例如site、page等，可以点击这里查看。

同时我们也会用到hexo的一些工具函数，例如partial，具体可以点击这里查看

到这里，我们主题的基本功能算是有了，至于是否美观，我们先不去考虑。让我们写两篇文章，测试一下我们的模板是否正常运行。

文章创作中…

写好文章后，我们只需要运行hexo server，就能在本地预览自己的博客的效果了。

我的博客要酱紫

到上一节，我们至少能让我们的站点正常显示出来，但是我们没有给它添加任何的样式或交互，作为一个程序猿来说，这是真心不爽的。

下面就要发挥我们的特长，将站点装饰一番了。

我们继续学步，参考一下landscape中的目录结构，我们可以惊奇的发现人家有个source目录，打开来看一下，要西，css、js、fancybox这不是一应俱全嘛。

那我们也可以在自己的主题目录下，同样建立一个source目录

garagekits
 |
 |--layout
 |     |--_partial
 |     |     |-- post_entry.ejs #主页文章列表中每项模板
 |     |
 |     |--index.ejs #网站主页模板，必不可少
 |     |--post.ejs # 每篇单独文章的模板
 |
 |--source
 |     |--css
 |     |     |--index.css
 |     |
 |     |--js
 |     |     |--script.js
 |     |

现在，一个主题最常见的基本部分已经初具规模了。万事俱备，只差你的idea了。现在开始就是你发挥你造物主的天才，构建一个与众不同的博客吧！！这酸爽，谁玩谁知道。
